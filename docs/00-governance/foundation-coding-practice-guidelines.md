# Foundation Coding Practice Guidelines

## Purpose
This note explains how coding practice should be handled during the Foundation phase of the Meridian project.

The goal is simple:
- keep momentum on real project work
- avoid unrelated coding detours
- use coding practice only when it directly supports the current Foundation task

## Scope
This note applies only to the Foundation phase.

Foundation includes:
- package boundaries
- shared primitives
- configuration discipline
- schema and migration discipline
- authentication shell
- correlation and error contract
- audit and outbox primitives
- file, storage, and antivirus safety seams
- local runtime and health checks

## Coding practice during Foundation

### Before a Foundation task
Usually, do no separate coding practice before starting.

Only do a very small warm-up if one of these feels weak:
- object or DTO mapping
- validation logic
- understanding the difference between unit tests and integration tests

### During a Foundation task
This is the main coding practice time.

Use the real Foundation work itself as the main practice.

Examples:
- clean object creation
- simple DTO mapping
- validation branching
- collection handling used in setup or tests
- test-writing basics
- clean code habits

### After a Foundation task
Only do very short reinforcement if needed.

Examples:
- one tiny mapping cleanup
- one tiny validation reinforcement
- one tiny test-writing reinforcement

Do not turn this into a long separate study block.

## What to skip for now
Do not spend Foundation time on:
- random LeetCode
- graph problems
- dynamic programming
- heap-heavy interview drills
- advanced concurrency exercises
- unrelated DSA sessions

These may matter later, but not for this phase.

## Working rule
Build first.
Use coding practice as support, not as a detour.

## Completion signal
This guideline is serving its purpose when Foundation work can move forward without repeated confusion about whether separate coding practice is needed before starting a task.
